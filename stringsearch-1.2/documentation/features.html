StringSearch - high-performance pattern matching algorithms in Java - documentation

Table of contents - 2. Features

The StringSearch library - while being reasonably small - attempts to provide a variety of high-performance pattern matching algorithms for all common data types.

All algorithms search in bytes, chars and Strings. If accessing the underlying char array in String through Reflection is allowed, the overhead of cloning the char array is avoided if the text to search exceeds a certain length. This length is based on the speed of Reflection in the Java Virtual Machine used. The length is discovered when the com.eaio.stringsearch.StringSearch class is loaded. As of Release 1, performance data from the current Java Virtual Machines from Sun, IBM, BEA and Transvirtual is included.

StringSearch also contains a stripped-down version with only three algorithms called StringSearch-mini that is useful if few space is available. StringSearch-mini contains the BNDM, BNDM Wildcards and Boyer, Moore, Horspool, Raita algorithms in a JAR archive of only 10 KB size. StringSearch-mini does not use the native library.

StringSearch contains a native library that further accelerates searching in bytes. Release 1 contains precompiled versions for Microsoft Windows (NativeSearch.dll) and Linux (libNativeSearch.so). To use the native library, set the java.library.path accordingly. If the native library is in lib/, invoke your application through java -jar app.jar -Djava.library.path=lib. If the standard StringSearch library is used, the usesNative method may be called to find out if an algorithm uses the native library. This method is not included in StringSearch-mini.

All implementations are thread-safe - one instance can be used by as many threads as required.
